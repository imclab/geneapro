Reports

All genealogical software include reports. These are views into your genealogical tree, showing various aspects.

Geneapro currently only includes a limited number of views. These are

  • A Pedigree view

    Given a person (often called the decujus), the pedigree shows a tree of all its ancestors up to a number of generations.

    Geneapro lets you configure the number of generations, and is comfortable showing all of them if you want. On my own genealogical tree, I have people up to the 21st generation (not all correctly verified yet!), and I can view the whole tree in my web browser, although that's sometimes a bit slow. But displaying 8 or 9 generations is no problem.

    Geneapro will also show you the children of the decujus.

    Clicking on any person in the tree will make that person the new decujus, and show his/her own tree.

    There are currently two implementations of the pedigree view. One is based on SVG, and is slower. It will soon be discarded. The second one is based on the HTML5 canvas, and works correctly in all modern browsers (Chrome, Firefox and Opera). Hasn't been tested on Internet Explorer 9.

    Since sources as so important in geneapro, it will add a check mark next to all dates and places for which at least one source is available.

    The styling used for each person (background color, text color, font,...) can be configured by creating Highlighting rules.

    Double-clicking on a person leads you to the Person view

  • A Fan chart

    Instead of displaying the ancestors as a tree, this report will show them in a circle. This is less frequently used, but is a much more compact way to see lots of generations in a single view.

    The same highlighting rules apply in the fanchart, although as seen in the screenshots it is also possible to use different color schemes, based for instance on the location of the person in the fanchart. This last colorscheme is less useful to investigate your data, but much nicer to print.

  • Quilts view

    Quilts view have been introduced a few years ago by Aviz Research, and are a great way to see a lot of people interactively. See the original paper. Geneapro does not provide the full functionality yet.

  • A statistics view

    This page shows the number of persons in your tree (not the total number in the database, which very often is irrelevant because people include information for person whose name seem to match, just in case it is useful later).

    For each generation, it is able to display the earliest birth date and latest known death date. These are often useful to detect inconsistencies in the database.

  • A Person view This view summarizes all knowns characteristics of a person, and all events to which she was part (either as the principal, or in any other role). When sources are available for these conclusions, hyper-links to the source view are displayed. This allows you to quickly see which conclusions need further sources.

    It allow shows the various persona that are linked together to represent a real world person, and will allow you to break these links when they are no longer valid.

    For each event or characteristic, the surety level of the assertion is displayed as a star ration. Note in this screenshot that the Birth event has a different surety scheme (these can be customized at will, so you can adapt to whatever your own conventions are). In the case of the birth, there is for instance only a single possible value for surety schemes (not very interesting...).

  • A List of persons view This view lists all known persons in the database. The table can be sorted on any of the columns, or can be filtered to see a subset of the persons. Clicking on a person will lead you to the Person view.
  • A Source view This view describes a source. It shows all representations (images) associated with the source. Each representation can be scrolled and zoomed. It is also possible to perform a few basic transformations on the images (convert to black-and-white,...) hopefully to make them more readable.
    In addition, the details of the source (name, repository,...) are displayed.
    Finally, it shows all facts and assertions that were deducted from this source. There are few genealogical software that show this, although this is a very convenient way to see in one glance whether all information was extracted from that source, or whether you might have missed some part of it when you analyzed it.

The long term goal is to make creating such reports easier (at least when dealing with exporting such reports). A good inspiration is LifeLines. Although a text based tool (and which cannot import my Gedcom file :-(), it is known for its good scriptability. One of its drawback is that the scripting language was designed 20 years ago, and is very low-level. Thanks to python, we can have a much more user-friendly language.

Geneapro will need to include a high-level API (ie hiding the database model), and then we can have reports exporting Books, Trees, List of people,...
